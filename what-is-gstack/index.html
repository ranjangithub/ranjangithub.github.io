What is gstack? Garry Tan's Claude Code Skill System Explained

The one-sentence answer, the philosophy behind it, and why it changes how Claude Code actually works.

The one-sentence answer

Definition

gstack is a directory of SKILL.md files that give Claude Code named, reusable workflows — stored in ~/.claude/skills/ — that persist across every session, every project, and every machine.

When you type /qa in Claude Code, Claude reads ~/.claude/skills/qa/SKILL.md and follows the workflow defined there. That file tells Claude exactly how to open your app in a headless browser, what to test, how to report bugs, and how to write fixes.

No magic. No separate API. No subscription. Just a file Claude reads and follows.

What problem it solves

Claude Code is stateless by design. Every session starts fresh. Every bug fix has to be re-explained. The third time you describe the same architecture to Claude is the moment most developers give up on getting consistent results.

The standard workaround is a CLAUDE.md file — project-specific instructions Claude reads at the start of each session. But CLAUDE.md is a flat document. It doesn't encode workflows. It doesn't know when to test vs when to ship vs when to stop and ask a product question.

What skills add

Skills are named, invocable workflows. /qa doesn't just remind Claude to test — it runs a structured testing protocol with a headless browser. /ship doesn't just remind Claude to commit — it runs tests, reads the diff, writes the PR, waits for CI, and merges. The skill is the workflow, not a note about the workflow.

Claude Code checks ~/.claude/skills/ automatically. When you type /qa, it finds qa/SKILL.md and treats the contents as instructions to follow. The skill files are plain Markdown — you can read them, fork them, and write your own.

The three things that make it different from prompting

1. Persistence. Prompts die with the session. Skills survive. Once gstack is installed, /qa is available in every Claude Code session, every project, forever.

2. Structure. A skill encodes a workflow — ordered steps, decision points, outputs. A prompt encodes a request. The difference is the difference between a checklist and a sticky note.

3. Composability. Skills call each other. /autoplan runs /plan-ceo-review, then /plan-design-review, then /plan-eng-review. You get a full review pipeline from one command.

Is it free?

Yes. gstack is fully open source on GitHub. No account required. No API calls to a separate service. You're paying for Claude Code (or using the free tier) and gstack runs inside it.
